> A terminal-aesthetic habit tracker with CLI look-and-feel, monospace fonts, 12 colour themes, GitHub-style contribution graphs, streaks, habit scoring, chains, and a full XP/levelling system.

## Features

### Core Tracking
- **Daily habit check-in** вҖ” boolean, count, and timer habits
- **Grace period** вҖ” completions logged before N:00 AM count for the previous day
- **Schedule-aware** вҖ” set habits to specific weekdays or `x per week`
- **Negative habits** вҖ” track things you want to stop

### Habit Intelligence
- **Habit Score (EMA)** вҖ” Loop Habit Tracker algorithm, converges to 100% after ~13 perfect days, visualised as a live bar `[в–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–Ҳв–‘в–‘] 82%`
- **14-day sparkline** вҖ” mini bar chart showing score momentum on each habit detail screen
- **Predictive insights** вҖ” streak projection, weakest habit of the week, best performance window, weekday vs weekend ratio
- **Habit correlations** вҖ” automatically surfaces which habits you tend to do together

### Streak System
- **Current & best streak** tracked per-habit and globally
- **Streak Shields** вҖ” earned at 7 / 14 / 30-day milestones (bronze / silver / gold), consume instead of breaking a streak
- **Habit Chains** вҖ” link habits that you always do in sequence; chain progress bar shows completion flow

### Achievements (38 total, 3 hidden)
Organised into 5 categories: **Consistency**, **Volume**, **Diversity**, **Time**, **Hidden**

Hidden achievements include рҹҰү *Night Owl* (5 late-night completions), рҹҢ… *Early Bird* (7 before-6am days), and рҹ’Ҝ *The Hundred* (100-day current streak).

### Navigation
- **Habits** вҖ” daily check-in with week strip and contribution graph
- **Stats** вҖ” overview, monthly, weekly, and per-habit breakdowns
- **+ Add** вҖ” create or edit habits with schedule, difficulty, reminders
- **Insights** вҖ” EMA scores, leaderboard (sort by streak / score / xp), predictive analysis, correlations
- **Profile** вҖ” theme picker, templates, achievements, chains, settings, export

## CI/CD

GitHub Actions workflow at `.github/workflows/flutter-build.yml`:

1. Sets up Java 17 + Flutter stable
2. Runs `flutter analyze` (non-blocking) and `flutter test`
3. Generates placeholder `google-services.json` for CI
4. Builds release APK with `--no-shrink`
5. Uploads APK as workflow artifact
6. Commits `app-release.apk` back to `main` branch
